About Me

I've always liked being creative and building things. Those things are usually built for the internet these days, though they used to mostly be built out of Lego. Ah, simpler times.

If you’ve ever played with Lego (and statistically you probably have as there are approximately 62 Lego bricks for every person on the planet today) you’ll know that it usually requires patience, problem solving and nimble fingers. Luckily for me these qualities translated nicely into a career web design and development.

My interest was sparked in school creating the simplest web pages you could imagine. But even these small projects gave me a glimpse at the true power that HTML and CSS possessed. We didn’t even touch on JavaScript!

It wasn’t until much later though that I was creating websites professionally. I had dabbled in video games development for a number of years but ultimately did not find it to be fulfilling. It was during this time however that I started working with a US-based start-up, Disa Technologies. Their website was in need of a spruce up and it was the perfect opportunity to dive headfirst back into web development. I taught myself everything I needed for that project by following tutorials, breaking things and doing a lot of late-night googling!

I have also worked on a few interesting side projects for friends and family that required getting familiar with WordPress and PHP.

Skills

Development

The core skills I use every day are, of course, HTML and CSS (compiled from SASS). These are the fundamental aspects of any website or web application. I do my best to keep up with recent industry trends by exploring CSS frameworks like Bootstrap, Materialize and Tailwind. The responsiveness and accessibility of websites across all devices is vital so I’m always expanding my knowledge of new specifications, like Flexbox and the CSS Grid.

I am adept at utilising JavaScript to bring life to my web projects. Sometimes I’ll rely on the speed and efficiency of JQuery to implement some small, simple interactions but I’m just as comfortable working in plain JavaScript. I also like to use Node.js behind the scenes when working with task runners, like Grunt and Gulp, that help to automate the development workflow.

PHP-based WordPress has featured in a couple of side projects I’ve worked on for friends and family. While I admire the streamlined setup process and ease-of-use of a standard WordPress website, I have also used PHP to add extra functionality to those projects in the form of custom page templates and navigation menus.

Like most I also use Git and Github as my version control system. This pairs nicely with my code editor of choice VS Code where I can see all of my recent changes, fix merge conflicts and stage commits without having to leave the editor.

Design

Figma is my favourite design tool. I use it for both graphic design and web design. I’ve also used it to create and maintain brand style guides. This includes the atomic elements of design like fonts and colors as well as modules like buttons, form elements and menus.

My approach to web design is to build wireframes of each of the required pages using the design elements established in the style guide and then prototype several variations of the designs by applying the brand styles until myself (and the client) are happy.
